github pages博客搭建

Posted on 2016-06-11

一、准备工作

1. github

博客搭建在github的github pages上，所以需要注册一个github账号，根据官方说明同时建一个与账户名同名的仓库。

注意：

  1. 仓库名称要和github上的用户昵称相同
    如果我的是昵称是rfish001那么仓库名称就必须rfish001.github.io
  2. 最好使用ssh模式clone到本地。
    方便平时生成博客

2. git

下载最新的git版本管理软件，并安装。用于推送本地生成的静态博客到github上。

3. node.js

HEXO静态生成博客需要用到node.js，所以需要安装该程序。安装完该程序后就可以使用npm命令了。

安装完该程序后由于软件源的原因，需要翻墙或者换成淘宝源。
替换为淘宝源：打开cmd输入如下命令$ npm install -g cnpm --registry=https://registry.npm.taobao.org之后使用cnpm命令代替npm命令

二、安装HEXO

1. 安装HEXO

在git中cd到任意文件夹，使用cnpm install -g hexo-cli安装HEXO。

注意：
该目录并不是用来存推送到github上的文件，而是用来生成需要推送上去的文件，而这个生成的文件是通过后面安装的hexo git 插件来完成的。

继续输入cnpm install hexo --save完成安装，验证HEXO安装是否成功输入hexo -v（查看HEXO版本）来间接验证。

注意：
以上命令均在git Bash中输入。CMD中应该也是可以的，但是没有git中的那么顺手，git中的命令很多延续了linux bash中的命令。

2. 初始化hexo

hexo init

3. 安装生成器

cnpm install

4. 运行HEXO

hexo s -g（s代表server ，表示启动服务，g表示generate生成博客），然后在浏览器中输入localhost:4000就可以看到博客界面了。但是现在还在本地的，我们需要推送到github pages上面去。

三、安装HEXO插件

1. 安装hexo git插件

因为推送到github上的文件并不是目前目录下的文件。需用到该插件。输入cnpm install hexo-deployer-git --save将完成插件的安装。

2. 配置hexo git插件需要的信息

五、域名绑定

1. 生成CNAME文件

根据github pages官方说明，需要在库的根目录下创建CNAME文件。但是如果直接在git网页上创建，下次生成发布博客的时候会被清理掉，但是我们可以在本地创建在/source下，在生成静态文件提交的时候，会被自动复制到根目录下。
